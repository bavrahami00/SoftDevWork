A Quick Guide to LaTeX


What is LaTeX?

It is a text formatter - something that takes in plaintext and spits out a pdf formatted with the commands that you give it. What commands? We'll get to that in a minute, but they're there to make life easier. LaTeX is WYSIWYM - What You See Is What You Mean - so you don't have to worry about the proper spacing or font or line breaks (if you give it the proper commands); it'll take of that for you.

Although revolutionary when first produced, in 1984, word processors such as Google Docs and Microsoft Word have caught up to it in the intervening years, so that it is no longer quite as useful as it once was, but there is one place where it is still an industry standard: mathematics. Since math has so many symbols and subscripts and superscripts, LaTeX is still used to write almost all papers. On arxiv.org, a scholarly archive, roughly 97% of all mathematical papers were written with LaTeX. And in addition, it has that lovely font, so you can use it even if it provides no other upgrade.


How to install LaTeX

On Linux, this can be done directly from the terminal: sudo apt install texlive

On Mac or Windows, this can be done by downloading the MiKTeX package.


There are also specialized editors that can be used for LaTeX such as TeXMaker or TeXStudio, but since LaTeX takes in plaintext, as previously mentioned, it can be done in ordinary text editors such as nano.

All LaTeX articles have the ending .tex, and then are converted to pdf either by a button on an editor, or by the commmand pdflatex <filename>, which then can be opened with your favorite pdf viewer.


So how does LateX work?

Any piece of code is either a piece of text that you want in the pdf or a command.

Commands are of the form \<command>[<optional parameters>]{<required parameters>} (with a very few exceptions), and they do one of two things: either they format the document in some way or they insert a special character not found on the keyboard.


LaTeX has two modes - normal mode and math mode. Normal mode is what you expect; what you write is put on the pdf, but it largely ignores whitespace, so you have to use a command to put multiple spaces between words or to force a new line. To create a new paragraph, leave an empty line in the code. In math mode, LaTeX assumes that everything that is not a command is a variable, and gives the text a "math font" so it looks professional.

There are also environments - commands of the form \begin{<environment>} ... \end{<environment>} which format the text inside it.

A List of LaTeX Commands

Command Mode Effect/Notes
\documentclass{<document type>} Preamble (before document) Required. Must be the first line of any document. The most common type of document is article.
\usepackage{<packagename>} Preamble If you want to use commands that are in some package that has not been installed yet, use this command.
\title{<title>}, \author{<author>}, \date{<date>} Preamble This will make the title/author/date of the paper being written be featured prominently on the page (once \maketitle is written).
\begin{document} ... \end{document} Environment Required. Everything that has to be written in the document has to be written inside here.
% All Creates a comment - everything else written on the line is ignored.
\maketitle Normal Makes the title according to the title/author date given
\section{<section name>}, \subsection{<section name>} Normal Creates a (numbered) section/subsection in the pdf, which is large and bolded.
\underline{<text>} Normal Underlines the text.
\emph{<text>} Normal Italicizes (or deitalicizes, if within another \emph) the text.
\begin{flushleft} ... \end{flushleft}, \begin{center} ... \end{center}, \begin{flushright} ... \end{flushright} Environment Aligns the text to the left, right, or center.
